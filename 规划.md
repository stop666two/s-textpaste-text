# S-TextPaste 规划文档 v1.0

## 项目定义

S-TextPaste 是一个绝密级零信任端到端加密文本分享服务。所有加解密在浏览器完成，服务端仅存储密文。

- 仓库：`https://github.com/stop666two/S-TextPaste`
- 部署：Cloudflare Workers + D1
- 一键部署按钮：README 中可用

## 核心特性

### 加密
- 浏览器端 Web Crypto API
- 强制密码保护，SHA-256 + MD5 链式 KDF
- 三重包络 AES-256-GCM 后量子加密
- HMAC-SHA-256 完整性校验
- 32字符随机ID防爆破 + 速率限制

### Markdown
- CodeMirror 6 编辑器，语法高亮，自动补全
- 实时分屏预览
- GFM / Mermaid / KaTeX / 代码高亮

### 生命周期
- 过期时间、阅后即焚、最大浏览次数、密码错误锁定

### UI/UX
- 暗黑模式、中英文切换、自定义短链接、安全性仪表盘

## 加密方案

### 密码派生密钥 (512-bit KDF, 12轮)
1. SHA-256(pw||salt) → 32字节
2. MD5(pw||salt) → 16字节
3. 链式迭代12轮，每轮 SHA-256 + MD5 XOR交织
4. 64字节输出 → 前32字节作AES-256密钥

### 后量子密钥 (1024-bit KDF-A, 24轮)
1. subSalt = SHA-256(salt||"PQ-A-v3")
2. 24轮 SHA主导 → MD5嵌套链式迭代
3. 128字节输出

### 后量子密钥 (1024-bit KDF-B, 24轮)
1. subSalt = SHA-256(salt||"PQ-B-v3")
2. 24轮 MD5主导 → SHA嵌套链式迭代 (与A完全逆向)
3. 128字节输出

### 三重包络加密
1. DEK(随机32字节) → AES-256-GCM加密明文
2. KDF-A(1024bit) → AES-256-GCM加密DEK
3. KDF-B(1024bit) → AES-256-GCM加密上一步结果
4. HMAC-SHA-256(integrityKey, 全载荷)

### 对称密钥模式
用户提供密钥 → 直接AES加密 → SHA-256+MD5推导PQ密钥 → 三重包络

### 非对称模式
RSA-OAEP公钥加密DEK → AES-256-GCM加密明文

## 数据库设计 (D1)

```sql
CREATE TABLE pastes (
  id TEXT PRIMARY KEY,
  mode TEXT NOT NULL,
  salt TEXT,
  encrypted_payload TEXT NOT NULL,
  hint TEXT DEFAULT '',
  delete_token_hash TEXT NOT NULL,
  expires_at INTEGER,
  max_views INTEGER DEFAULT -1,
  view_count INTEGER DEFAULT 0,
  burn_after_read INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  pubkey_fingerprint TEXT
);
```

## API

| 方法 | 路径 | 请求 | 返回 |
|------|------|------|------|
| POST | /api/paste | mode, encrypted_payload, ... | id, delete_token |
| GET | /api/paste/:id | - | encrypted_payload, view_count, ... |
| POST | /api/paste/:id/view | - | success |
| DELETE | /api/paste/:id | X-Delete-Token | success |
